# Dual Agent Orchestrator 测试场景设计

测试日期：2026-05-30
测试目的：全面评估项目当前能力，验证 demo 就绪状态

## 测试场景设计

### 场景 1：技术研究任务（Research + Evidence）
**任务描述**：研究 2026 年最新的 AI Agent 框架对比，生成 markdown 报告

**测试目标**：
- 验证 web_search 工具能力
- 验证 url_fetch 抓取能力
- 验证多轮搜索与证据收集
- 验证 write_file 输出能力
- 验证 planner 的研究规划能力

**预期输出**：
- `reports/ai-agent-frameworks-2026.md`
- 包含至少 3 个主流框架对比
- 包含来源链接

**成功标准**：
- ✅ 成功搜索并抓取多个来源
- ✅ 生成结构化 markdown 报告
- ✅ 报告内容真实可信
- ✅ 完整的 workflow 事件流

---

### 场景 2：数据处理任务（File Ops + Data Analysis）
**任务描述**：读取项目的 package.json，分析依赖项，生成依赖分析报告

**测试目标**：
- 验证 read_file 能力
- 验证 parse_json 能力
- 验证数据分析逻辑
- 验证结构化输出

**预期输出**：
- `reports/dependency-analysis.json`
- 包含依赖统计、版本信息

**成功标准**：
- ✅ 正确读取并解析 JSON
- ✅ 生成准确的统计数据
- ✅ 输出格式正确

---

### 场景 3：代码生成任务（Code + File Ops）
**任务描述**：生成一个简单的 TypeScript 工具函数库（日期格式化、字符串处理）

**测试目标**：
- 验证代码生成能力
- 验证多文件创建
- 验证代码质量

**预期输出**：
- `src/utils/date-utils.ts`
- `src/utils/string-utils.ts`
- `src/utils/index.ts`

**成功标准**：
- ✅ 生成可编译的 TypeScript 代码
- ✅ 包含类型定义
- ✅ 代码结构合理

---

### 场景 4：混合任务（Research + Code + File Ops）
**任务描述**：研究 GitHub API 使用方法，然后生成一个调用示例脚本

**测试目标**：
- 验证跨类型任务协同
- 验证 planner 的任务分解能力
- 验证 executor 切换能力

**预期输出**：
- `examples/github-api-demo.ts`
- `examples/README.md`

**成功标准**：
- ✅ 研究阶段获取准确信息
- ✅ 代码示例可运行
- ✅ 文档说明清晰

---

### 场景 5：长任务与恢复（Job Control + Resume）
**任务描述**：创建一个需要多步骤的复杂任务，测试暂停和恢复能力

**测试目标**：
- 验证 Job 控制面
- 验证 async 执行
- 验证事件流
- 验证 resume 能力

**预期行为**：
- 创建 async job
- 通过 /stream 观察进度
- 测试 cancel/retry/resume

**成功标准**：
- ✅ Job 正确创建并持久化
- ✅ 事件流实时推送
- ✅ 控制操作生效
- ✅ Timeline 可视化正常

---
