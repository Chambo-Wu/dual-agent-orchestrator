## 1. 总体结论

> 2026-05-29 同步：本清单最初用于拆首轮实施任务；当前代码已把 Phase 1-5 的主链路基本落完，包括 outcome / reflection / proposal / auditor / validation / accept-reject / limited automation。后续工作重点已从“是否实现”转为“提升 proposal 质量、增强真实 replay、补运营视图与文档治理”。

这条线建议按 5 个阶段推进，而不是一次性做完整自动自进化：

1. Phase 1
   - `skill outcome + reflection`
2. Phase 2
   - `proposal generation`
3. Phase 3
   - `auditor gate`
4. Phase 4
   - `deployment validation`
5. Phase 5
   - `limited automation`

原推荐实际排期：

- 第一轮只做 Phase 1
- 第二轮做 Phase 2 + Phase 3
- 第三轮再做 Phase 4
- Phase 5 最后评估是否值得打开

当前实际状态：

- Phase 1 已落地：skill outcome summary、reflection classifier、reflection events / timeline / API。
- Phase 2 已落地：proposal store、candidate 目录、proposal API、candidate materialization。
- Phase 3 已落地：deterministic auditor gate 与关键安全检查。
- Phase 4 已落地：candidate-aware validation、same-input comparison、risk-tier hard gate、deterministic isolated manifest replay、replay job/events。
- Phase 5 已落地受控版本：配置化 auto reflect / propose / audit / validate / accept，并受 risk tier automation ceiling 与 `autoAcceptReady` 约束。

## 2. 推进原则

- 默认不自动修改 `skills/`
- 默认不自动 accept
- 所有候选 patch 先落 `runtime/skill-evolution/`
- 所有关键状态进入 `job / events / timeline / health`
- 首批只针对低风险 skill
  - `find.official_sources`
  - `find.workspace_files`
