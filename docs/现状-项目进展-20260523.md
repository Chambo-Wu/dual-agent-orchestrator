### Gap 2: Resume 端点

**改动文件：**
- `src/job-store.ts` — `JobControlState` 增加 `resumedAt`, `resumedToJobId`, `resumeOf`
- `src/index.ts` — 新增 `POST /v1/jobs/:id/resume` 路由和 `handleResumeJob`

**API 说明：**
- `POST /v1/jobs/:id/resume` — 重新执行一个 blocked/failed 的 job
- 创建新 job 并通过 control metadata 链接原 job（与 retry 模式一致）
- 原 job 的 `control.resumedToJobId` 指向新 job，新 job 的 `control.resumeOf` 指向原 job
- 返回 400 如果 job 已完成，返回 409 如果 job 正在运行

### Gap 4: Verification Layer

**改动文件：**
- `src/verification.ts`（新建）— 5 个系统级校验器
- `src/index.ts` — 集成到 `executeTaskGoal`，任务完成后自动运行校验

**校验器：**

| 校验器 | 说明 |
|---|---|
| `FileExistsVerifier` | 检查 file 类型 artifact 指向的文件是否存在 |
| `SchemaCheckVerifier` | 检查 JSON 类型 artifact 是否合法 JSON |
| `ArtifactPresenceVerifier` | 检查是否至少有一个非空 artifact |
| `GitDiffVerifier` | 检查工作区是否有文件变更 |
| `UrlReachableVerifier` | URL 可达性检查（当前跳过，预留接口） |

**集成方式：** 在 `executeTaskGoal` 中，任务完成后自动运行所有校验器。如果校验失败，`job.verified` 设为 `false`。
