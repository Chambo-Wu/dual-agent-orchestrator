# 【历史归档】项目进展 2026-05-25

## 今日总览

今天的优化改造集中在两条主线：

1. 完成 `workflow_plan` 的 Milestone C 收口，包括 schema、runtime、replan、事件与测试闭环。
2. 把前端时间线里的 DAG 面板从“任务列表 + depends_on 文本”升级为真正的 dependency graph 视图，并补齐 replan 联动交互。

这意味着项目当前已经不只是“能执行 workflow”，而是同时具备了：

- 显式 `workflow_plan` 解析与校验
- workflow runtime 执行与 replan

## 今日完成的改造

### 1. `workflow_plan` schema 与 runtime 落地

新增并接入了以下能力：

- `src/workflow-plan.ts`
  - 解析与校验 `workflow_plan`
  - 约束 task id、依赖关系、支持的 task type、`retry_policy`、`finish_when`、`replan_policy`
- `src/workflow-runtime.ts`
  - 按依赖顺序执行 task
  - 支持 `delegate / synthesize / verify / approval` 等执行语义
  - 支持 `finish_when` 的真实运行时行为，而不是仅停留在 schema
  - 支持 runtime replan，并保留 superseded workflow 历史
- `src/workflow-graph.ts`
  - 从 workflow 执行结果构建显式 graph 视图数据
  - 输出 active / superseded workflow lane 与 replan history

配套修改还涉及：

- `src/index.ts`
  - job response 中补充 workflow summary / dag / replan history
- `src/job-store.ts`
  - 配合持久化结构扩展
- `src/types.ts`
  - 增补 workflow plan / graph / runtime 相关类型
- `src/workflow-contract.ts`
  - 补齐前后端消费 workflow 数据时的契约

### 2. DAG 面板升级为真正的 dependency graph

`src/timeline.ts` 里的 DAG 面板已经从 lane/card 列表升级为 graph 视图：

- 不再只靠 `Depends on: ...` 文本读结构
- 先基于显式 `depends_on` 计算 task stage
- 再按 stage 自动分列布局
- 使用 SVG 直接绘制 task 之间的 dependency edge
- `superseded workflow` 继续保留独立 lane，并使用不同颜色和边样式

关键点是：

- 前台使用的仍然是显式 `workflow_graph` schema
- 没有退回到“从 task list 猜结构”的弱约束做法

### 3. replan history 与 graph 联动增强

今天还补齐了 graph 与 replan history 的交互层：

- history item 点击可聚焦对应 workflow lane
- 同一条 replan 记录可在 `superseded lane` 和 `replacement lane` 之间切换
- 提供 `Show all lanes` 清除聚焦
- lane 自身也支持点击聚焦 / 取消聚焦
- 当前聚焦状态会明确显示：
  - `Focused: superseded lane`
  - `Focused: replacement lane`
- 同时给出下一步提示文案，减少“第二次点击会发生什么”的猜测成本

### 4. graph 交互基础设施补齐

为了支持后续更深的前端能力，页面里已经补齐稳定 DOM 标识：

- lane: `data-workflow-id`
- task card: `data-task-id`、`data-workflow-id`、`data-depends-on`
- edge: `data-from`、`data-to`、`data-workflow-id`
- replan history item:
  - `data-superseded-workflow-id`
  - `data-replacement-workflow-id`

并新增了两类前端交互：

- 当前任务高亮：`is-current-task`
- hover 依赖链高亮：hover 某节点时，同时高亮 upstream / downstream 节点和边
