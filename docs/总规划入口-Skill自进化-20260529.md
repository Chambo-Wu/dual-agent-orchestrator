## 2. 当前判断

### 2.1 已完成前置条件

当前仓库已经具备 skill 自进化所需的关键基础能力：

- `skill-registry`
- `skill-installer`
- `skill-runtime`
- builtin skill manifests
- `candidateSkills / selectedSkill`
- skill install lifecycle events
- skill-aware timeline detail
- skill-aware verification

这意味着：

> skill 自进化现在不是从零开始，而是建立在“已有 skill 主链”之上的增强层。

### 2.2 已完成的 Skill 自进化 v1 主链

当前 skill 自进化已经不是“从 outcome/reflection 开始实现”的阶段。已完成主链包括：

- skill outcome summary
- reflection classifier
- evolve proposal
- proposal store
- candidate materialization
- auditor gate v1
- deployment validation v1/v1.5
- audit reports / validation reports
- accept / reject
- rollback snapshot
- limited automation
- risk tier automation ceiling
- proposal detail / validation summary / dashboard 基础摘要

### 2.3 当前真正未完成或需深化的内容

当前未完成项主要是 v2 深化与治理能力：

- Proposal Generator v2：高质量结构化 rewrite、fixtures、candidate content 断言。
- Deployment Validation v2：真实 candidate runtime replay、task-level replay payload、更强 acceptance contract。
- Observability / Ops View v2：proposal queue、accepted history、rollback guide、funnel、aging、stuck-state。
- Auditor Gate v2：risk-tier profile、cross-file consistency、段落职责审计、verification diff audit。
- `SKILL.md` 双结构治理：live skills inventory、check 脚本、分批迁移。
- Automation Risk Tiering v2：动态风险信号、failure-rate downgrade、replay stability score。

## 3. 目标定义

`Skill Evolve Workflow` 的目标不是让 Agent 直接改写生产 skill，而是建立一个受控闭环：

```text
selected skill
  -> real job outcome
  -> reflection
  -> proposal
  -> auditor
  -> deployment validation
  -> accept / reject
```

一句话概括：

> 先做“可审计的 skill 提案系统”，再逐步演进为“受控自动改进系统”。

### Phase 3：Auditor

状态：done

- deterministic isolated manifest replay
- replay job / events
- `autoAcceptReady`

### Phase 5：有限自动化

状态：done

- risk tier automation ceiling
- high / medium / low ceiling block behavior
