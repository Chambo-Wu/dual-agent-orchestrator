## 2. 当前判断

### 已完成主链

- GoalMode：
  - durable goal object
  - `/v1/goals` create/list/get
  - `run-next`
  - `retry / resume / review`
  - final review 手动触发链路
  - goal dashboard / timeline / events 基础能力
- Skill 自进化：
  - outcome / reflection
  - proposal generation v1
  - candidate materialization
  - auditor gate v1
  - deployment validation v1/v1.5
  - accept / reject / rollback snapshot
  - limited automation + risk tier ceiling
  - proposal detail / validation summary / dashboard 基础摘要

### 仍需完成或补强

- GoalMode：
  - 显式传入 `tasks` 时的 `large_check` 语义需要收口。
  - `large_check` 需要 integration/API contract 覆盖。
  - goal observability endpoint 需要 contract 覆盖。
  - final review 是手动还是自动触发，需要文档明确。
  - GoalMode docs 仍把已完成主链标成 todo，需要同步。
- Skill 自进化：
  - Proposal Generator 仍是 v1，未达到高质量结构化 rewrite。
  - Deployment Validation 仍未做到完整真实 candidate runtime replay。
  - Observability / Control Plane 缺 proposal queue / accepted history / rollback guide / funnel / aging。
  - Auditor Gate 缺 v2 的 risk-tier profile、cross-file consistency、段落职责审计。
  - `SKILL.md` 双结构治理缺 live skills inventory、lint/check、分批迁移。
  - Automation Risk Tiering 缺动态风险信号和运行稳定性降级。

## 3. 总体实施策略

采用“主线 + 并行支线 + 最后收口”：

1. 先修正 GoalMode 与 docs 的事实漂移，避免继续基于过时清单排期。
2. 并行推进 Skill 自进化中互不冲突的 v2 基础切片。
3. Deployment Validation v2 与 Proposal Generator v2 是主线，其他治理能力作为支线支撑。
4. Automation Risk Tiering v2 放到最后，因为它依赖 validation、auditor、observability 的成熟信号。
