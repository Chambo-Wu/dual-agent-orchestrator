# 【已完成】快速启动与测试指南

## 启动服务

### 方式 1：前台启动（推荐用于开发）
```powershell
npm run serve
```

### 方式 2：后台启动（Windows）
```powershell
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd D:\Android\dual-agent-orchestrator; npm run serve"
```

### 方式 3：使用已有脚本
```powershell
npm run serve:restart:9898
```

### 方式 4：Electron 桌面主界面
```powershell
npm run desktop
```

桌面端行为：
- 启动时读取 `config/config.yml`，并以 `DUAL_AGENT_CONFIG=config/config.yml` 启动本地 API 服务
- 可配置模型 `id`、`API URL`、`API Key`、模型 ID、timeout、tokens、temperature
- 可设定模型角色：`planner`、`executor`、`worker`、`verifier`、`synthesizer`、`planner_proxy`
- 可配置模型路由，并直接执行 `task` / `team` 任务
- 已聚合 Jobs Dashboard、Goals Dashboard、Skill Ops、Health、Timeline、Events
- 在桌面端保存模型/角色配置时，会写回 `config/config.yml`，并保留原有 `policy`、`search`、`skills`、`skill_evolution`、`goal_mode` 等非模型配置段

## 验证服务状态

### 1. 检查健康状态
```bash
curl http://127.0.0.1:9898/health
```

### 2. 访问 Dashboard
浏览器打开：`http://127.0.0.1:9898/jobs/dashboard`

### 3. 访问 Goals Dashboard
浏览器打开：`http://127.0.0.1:9898/goals/dashboard`

### 4. 验证配置文件
```bash
node --enable-source-maps dist/index.js config validate config/config.yml
```

当前桌面端与服务端应共同使用 `config/config.yml`。如果发现加载到样例模型，先检查 Electron 服务日志中的 `DUAL_AGENT_CONFIG` 路径。

## HTTP API 测试

### 创建 Job（异步）
```bash
curl -X POST http://127.0.0.1:9898/v1/jobs \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer dual-agent-local" \
```

## 常见问题排查

### 问题 1：服务启动失败
**检查**：
```bash
npm run doctor
```

**可能原因**：
- 端口 9898 被占用
- 模型服务未启动
- 配置文件错误

### 问题 2：任务执行失败
**查看日志**：
```bash
ls -lt runtime/logs/*.jsonl | head -1 | awk '{print $NF}' | xargs cat
```

**查看 Job 记录**：
```bash
ls -lt runtime/jobs/*.json | head -1 | awk '{print $NF}' | xargs cat
```

### 问题 3：Shell 命令失败
**原因**：PowerShell/cmd 兼容性问题

**解决**：系统会自动重试，通常第 2-3 次会成功

## 性能监控

### 查看最近的 Job
```bash
curl http://127.0.0.1:9898/v1/jobs?limit=10 \
  -H "Authorization: Bearer dual-agent-local"
```

### 查看运行时配置
```bash
curl http://127.0.0.1:9898/v1/models \
  -H "Authorization: Bearer dual-agent-local"
```

### 查看 Electron 桌面端实际路由
桌面端“模型”页的路由列表会映射到 `/v1/models` 暴露的 `model_route`。执行任务时，请确认所选路由与预期的 planner / executor 组合一致。
