# 快速启动与测试指南

## 启动服务

### 方式 1：前台启动（推荐用于开发）
```powershell
npm run serve
```

### 方式 2：后台启动（Windows）
```powershell
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd D:\Android\dual-agent-orchestrator; npm run serve"
```

### 方式 3：使用已有脚本
```powershell
npm run serve:restart:9898
```

## HTTP API 测试

### 创建 Job（异步）
```bash
curl -X POST http://127.0.0.1:9898/v1/jobs \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer dual-agent-local" \
  -d '{"goal": "创建一个测试文件", "mode": "task"}'
```

### 查看 Job 列表
```bash
curl http://127.0.0.1:9898/v1/jobs \
  -H "Authorization: Bearer dual-agent-local"
```

### 查看 Job 详情
```bash
curl http://127.0.0.1:9898/v1/jobs/{job_id} \
  -H "Authorization: Bearer dual-agent-local"
```

### 订阅事件流
```bash
curl http://127.0.0.1:9898/v1/jobs/{job_id}/stream \
  -H "Authorization: Bearer dual-agent-local"
```

## 常见问题排查

### 问题 1：服务启动失败
**检查**：
```bash
npm run doctor
```

**可能原因**：
- 端口 9898 被占用
- 模型服务未启动
- 配置文件错误

### 问题 2：任务执行失败
**查看日志**：
```bash
ls -lt runtime/logs/*.jsonl | head -1 | awk '{print $NF}' | xargs cat
```

**查看 Job 记录**：
```bash
ls -lt runtime/jobs/*.json | head -1 | awk '{print $NF}' | xargs cat
```

### 问题 3：Shell 命令失败
**原因**：PowerShell/cmd 兼容性问题

**解决**：系统会自动重试，通常第 2-3 次会成功
